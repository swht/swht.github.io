一、git本地操作

git特点
  • 分布式，svn(集中式)
  • 回到过去，回到未来(更改状态点)
  • 多端共享
  • 解决冲突(冲突被显示出来,但是解决需要开发者手动去处理)
  • 控制版本
安装git

需要选择在cmd命令下使用unix简单命令

cd

charge dir 改变目录

pwd

打印当前工作目录，print working dir

配置git config

如果没有配置过git是不能进行提交操作的

初始化git
git有三个区
  • 工作区
  • 暂存区/缓存区
  • 历史区/版本库
查看提交版本

git log 命令是只能查看当期那版本以前的历史记录

查看所有的操作历史
代码回滚
一步提交
  • 缺点：如果有文件没有在历史提交过，则该命令提交失败
在历史区中查询关键字
代码比较

  • 工作区和暂存区

  • 暂存区和历史区

  • 工作区和历史区

    git branch

    git branch dev //创建本地分支
    git checkout dev //切换分支
    git checkout -b dev //创建并切换到新的分支

    git branch -D dev //删除指定分支

    git merge dev //在主分支，合并dev分支

    只能手动确认提交

    git init

    touch .gitignore

    .idea
node_modules
.DS_Store
bower_components

    git remote add origin

    git remote -v

    git remote rm 仓库别名

    git push origin master -u //-u 下次默认使用origin远程仓库

    git push origin master -f

    git push origin gh-pages
    ```
